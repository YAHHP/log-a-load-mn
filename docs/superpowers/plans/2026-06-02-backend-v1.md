# Backend V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a local Backend V1 that turns the Log A Load prototype into a testable event operating system with admin auth, records, ticket capacity checks, payment-link plumbing, exports, webhook intake, and email notification outbox.

**Architecture:** Keep the Vite frontend static-friendly while adding an Express API in `server/`. The API stores prototype records in `server/data/local-db.json`, guarded by a server-side admin token, so it can later move to a managed database without redesigning the public UI.

**Tech Stack:** React/Vite frontend, Express API, Node built-in test runner, JSON datastore, HMAC admin session token, hosted PayPal/Stripe/Venmo payment-link handoff.
